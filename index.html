Interfaces.py

Определяет абстрактные интерфейсы взаимодействия для компонентов ввода (клавиатура, события) и вывода (отрисовка, тайминги), реализуя принцип разделения интерфейсов (ISP).

IInputProvider.poll_events()
str

Опрашивает системные события фреймворка. Возвращает строковые директивы управления жизненным циклом приложения, такие как "quit", "reset" или "continue".

IInputProvider.get_actions()
dict[str, bool]

Возвращает текущую карту активных игровых команд управления (состояние кнопок "forward", "brake", "left", "right", "nitro") в виде логических значений.

IRenderer.setup(game)
None

Выполняет первоначальную конфигурацию графического окружения, генерирует тяжелые статические ресурсы (например, геометрию трассы) перед запуском основного цикла.

IRenderer.render(game)
None

Основной метод визуализации. Принимает ссылку на объект Game, считывает пространственные координаты игровых сущностей и отрисовывает обновленный кадр на экране.

IRenderer.tick()
float

Контролирует внутреннюю частоту обновления кадров (FPS) графической подсистемы и возвращает точное количество секунд, прошедшее с предыдущего вызова кадра (delta time).

Car.py

Содержит независимую математическую и физическую модель транспортного средства, абстрагированную от любых графических библиотек.

MathVector2.__init__(x, y)
MathVector2

Создает экземпляр двумерного вектора с плавающей точкой для хранения декартовых координат позиций и скоростей.

MathVector2.length()
float

Вычисляет евклидову длину (модуль) текущего вектора по теореме Пифагора.

MathVector2.scale_to_length(target_length)
None

Нормализует вектор и масштабирует его до строго заданной величины target_length без изменения текущего направления.

MathVector2.lerp(other, t)
MathVector2

Производит линейную интерполяцию между текущим вектором и вектором other с коэффициентом сглаживания t.

Car.__init__(x, y)
Car

Инициализирует физические параметры автомобиля, устанавливая начальный вектор положения, нулевой вектор скорости и базовый угол разворота.

Car.reset(x, y)
None

Мгновенно сбрасывает пространственные характеристики машины (координаты, скорость, направление) в исходное состояние.

Car.update(dt, actions)
None

Рассчитывает перемещение, угловое ускорение, линейную тягу двигателя, эффект торможения, импульс нитро-ускорителя и силу лобового сопротивления воздуха за квант времени dt.

Game.py

Управляет глобальным состоянием игрового мира, хранит структуры данных активных сущностей и агрегирует логические компоненты.

Game.__init__()
Game

Конструирует игровой контекст и создает физический объект управляемого игроком автомобиля на стартовой позиции.

Game.update(dt, actions)
None

Выполняет итерационное обновление всей внутренней симуляции мира, перенаправляя квант времени и вектор команд в нижележащие слои логики.

Game.reset()
None

Запускает процедуру восстановления первоначального состояния для всех активных объектов симуляции внутри игрового пространства.

Engine.py

Основной управляющий оркестратор, связывающий воедино игровую логику, ввод и отрисовку в бесконечном тактовом цикле.

Engine.__init__(game, renderer, input_provider)
Engine

Инжектирует необходимые зависимости через абстрактные типы интерфейсов, изолируя ядро приложения от конкретных платформ реализации.

Engine.run()
None

Запускает и поддерживает непрерывный игровой цикл. На каждом шаге синхронно обрабатывает прерывания ввода, обновляет физические расчеты, рендерит графику и запрашивает вычисление дельты времени.

Gapi.py

Графическая и интерактивная реализация интерфейсов на базе фреймворка Pygame, адаптированная под десктопные операционные системы.

Gapi.__init__()
Gapi

Инициализирует графические подсистемы, конфигурирует экранную матрицу дисплея, системные таймеры, структуры виртуальной камеры и кэширует шрифты.

Gapi._build_car_image()
Surface

Внутренний процедурный генератор графики. Формирует растровое изображение автомобиля со стеклами и скруглениями на выделенной текстурной поверхности.

Gapi.setup(game)
None

Реализация интерфейса. Вызывает процедурную генерацию карты трассы и кэширует ее для оптимизации последующей отрисовки.

Gapi.poll_events()
str

Реализация интерфейса. Считывает прерывания и сигналы от операционной системы из очереди событий (закрытие окна, нажатие клавиш Esc/R).

Gapi.get_actions()
dict[str, bool]

Реализация интерфейса. Сканирует низкоуровневые скан-коды физически зажатых клавиш клавиатуры и маппит их в логическую структуру действий.

Gapi.render(game)
None

Реализация интерфейса. Синхронизирует положение камеры, заполняет фон, выполняет блиттинг текстуры трассы и машины с учетом экранного смещения, выводит отладочный UI и переворачивает графический буфер.

Gapi._draw_car(player)
None

Рассчитывает матрицу вращения изображения машины в зависимости от ее угла движения и проецирует центр полученного спрайта в экранные координаты.

Gapi.tick()
float

Реализация интерфейса. Вызывает задержку кадра для фиксации частоты обновления и возвращает актуальный интервал времени обработки.

Gapi._update_camera(target)
None

Обновляет внутренние координаты двухмерного вектора следования камеры, центрируя область видимости экрана на переданной координатной точке машины.

Gapi._build_track()
tuple

Генерирует массив путевых точек кольцевого автодрома по тригонометрическим кривым, вычерчивает на холсте подложку асфальтового полотна и рассчитывает габариты текстуры.

main.py

Основная точка входа в программный комплекс, выполняющая композицию объектов и запуск приложения.

Блок __main__
Выполнение

Создает конкретный графический API (Gapi), инициализирует доменную область (Game), связывает их внутри диспетчера Engine и передает управление в главный исполнительный поток.
